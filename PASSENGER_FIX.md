# CRITICAL: Fix for Hostinger/Passenger Conflict

The `.htaccess` file you see on your server is automatically generated by Hostinger's "Node.js Setup" (Passenger). **It is conflicting with your manual PM2 setup.**

That configuration tries to run a file named `server.js` (which didn't exist) and intercepts all traffic, preventing your running app (on port 3000) from receiving requests correctly.

## Solution 1: Use PM2 (Recommended)

If you want to use the `pm2` setup we prepared (which is more robust):

1.  **Edit the `.htaccess` file on the server** (you are already in `nano`).
2.  **DELETE ALL** the lines starting with `Passenger...`.
3.  **PASTE** the following configuration instead:

    ```apache
    Options -MultiViews -Indexes

    RewriteEngine On
    ProxyPreserveHost On

    # Force HTTPS
    RewriteCond %{HTTPS} off
    RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

    # Proxy EVERYTHING to Next.js running on port 3000
    RewriteRule ^(.*)$ http://127.0.0.1:3000/$1 [P,L]
    ```

4.  **Save and Exit** (`Ctrl+O`, `Enter`, `Ctrl+X`).
5.  **Restart PM2**: `pm2 restart geomoney`

## Solution 2: Use Hostinger's Node.js Manager (Passenger)

If you prefer to use the Hostinger UI and NOT use PM2:

1.  I have created a `server.js` file in your project root. **Upload this file to your server.**
2.  In Hostinger's Node.js settings, ensure "Startup File" is set to `server.js`.
3.  Keep the `.htaccess` as it is (Passenger will manage it).
4.  **Stop PM2**: `pm2 stop geomoney` (you can't run both).
5.  Click "Restart" in the Hostinger Node.js dashboard.

**I strongly recommend Solution 1** because it gives you more control and avoids issues with Passenger's static file handling.
